# 2026-05-19

## What I worked on

Fixed two separate issues in wandb/weave on the same day.

**Fix 1 — Recursive OpenAPI schema (PR #6871):**
Weave describes its HTTP API using an OpenAPI JSON file. One of the types in it, `LiteralOperation`, had a field that pointed back to itself — like a dictionary definition that says "a word means... that word." When Mintlify (the docs generator) tried to render this, it went into an infinite loop.

The fix was to break the loop by replacing the self-references with open-ended types (`additionalProperties: true`, `items: {}`). These mean "can be any object" or "can be any array" — slightly less precise, but finite and renderable.

**Fix 2 — Stale error message (PR #6938):**
When a user names a Weave object with invalid characters (e.g. a slash), the validation code throws an error. That error used to say "please upgrade to `>0.51.0`" — but Weave is now at `0.52+`, so users on the current version see an instruction that's already out of date. Replaced it with a clear rule: "Object names must only contain alphanumeric characters, dashes, underscores, and periods."

**Process lesson — PR splitting:**
I accidentally put both fixes in one PR (#6888). A W&B reviewer noticed and asked to split them. Learned how to use `git cherry-pick` — take one specific commit and apply it to a fresh branch, leaving everything else behind. Closed #6888 and opened clean #6938 with only the validation fix.
