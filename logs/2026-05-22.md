**Fix 1 — Daily log cron not pushing:**
The cron job was running every morning, creating the log file, committing it locally — but failing silently on the push. The error was `fatal: could not read Username for 'https://github.com': Device not configured`.

The reason: HTTPS git remotes need a username and password (or token). When you push from a terminal, your credential manager handles this invisibly. But cron runs in a headless environment with no interactive session — no credential manager, no prompt, nothing. So the push just failed every day.

The fix: switch the remote from HTTPS to SSH. SSH uses a key file instead of a password, so it works in cron without any user interaction. Added the SSH public key to GitHub so it recognizes the machine.

After fixing, pushed all the backed-up local commits (May 17–21 logs were on disk but never made it to GitHub).

**Fix 2 — ragas PR going to wrong repo:**
When trying to submit the ragas PR, it kept going to `vibrantlabsai/ragas` instead of `explodinggradients/ragas`. After investigation: `explodinggradients` renamed their GitHub organization to `vibrantlabsai`. Same repo, same code, just a different URL. So the PR was going to the right place all along — just under a new name.

Lesson: always verify a fork's parent with `gh repo view <fork> --json parent` before opening a PR.
