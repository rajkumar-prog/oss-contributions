# 2026-05-17

## What I worked on

Fixed a crash in the openai/evals framework — PR #1664.

OpenAI's `evals` tool runs AI models against test questions and produces a report including how many tokens were used (tokens = the "words" an AI processes, and you pay per token). The code that summed up token usage assumed the values coming from the API were plain numbers. But OpenAI recently changed their API — instead of returning `42`, they now return a `CompletionTokensDetails` object. Python crashed with `TypeError` when the code tried to treat that object as a number.

The fix was a one-line change: added `int()` casting and a `None` guard so it handles both the old plain-number format and the new object format. The `int()` function works here because OpenAI's object class defines how to convert itself to an integer.

**Deep note:** This is a very common issue in software — you depend on an external API, that API changes the shape of its data, and your code silently breaks. The safe habit is always cast and guard external values before doing arithmetic on them.
